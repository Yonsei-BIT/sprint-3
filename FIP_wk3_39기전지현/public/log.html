06 · Process

08 · Iterations
수정 과정
시도 → 실패 → 수정의 반복
BUG
API Key 프론트 노출
→ Netlify Functions 백엔드 구조로 전환, 서버사이드에서만 API 호출
BUG
iOS Safari 카메라 "string did not match" 에러
→ 5단계 폴백 처리 (고화질 → 720p → 해상도 제약 없이 → 구형 방식 → any)
BUG
카메라 2배 줌으로 찍힘
→ object-fit: cover → contain 변경, 해상도 제약 제거
BUG
JSON 파싱 깨짐 + 504 timeout
→ max_tokens 2048→4096, 모델 sonnet→haiku, netlify 타임아웃 26초 설정
FIX
단일 페이지만 분석됨
→ 모든 촬영 페이지를 배열로 API에 전달, 멀티페이지 종합 분석
FIX
액션카드가 "3페이지 읽으세요" 반환
→ 프롬프트에 "페이지 참조 금지, 실제 내용 직접 추출" 명시
FIX
분석 후 자동 스크롤 (원하지 않는 동작)
→ 자동 스크롤 제거, 분석 버튼 하단 고정(sticky)으로 변경

12 · Learnings
새로 배운 AI 활용 방식
👁
Vision API = 별도 OCR 불필요이미지를 Base64로 넘기면 Claude가 텍스트+표+그림까지 한번에 이해
✍️
프롬프트가 곧 UX"페이지 참조 금지", "2줄 이내", "원문 복사 금지" 같은 제약이 실제 사용자 경험을 바꿈
🎯
구조화된 JSON 출력으로 UX 제어자유 텍스트 대신 JSON schema를 강제 → 프론트에서 표/카드로 시각화 가능
⚖️
모델 선택 = 속도/비용/품질 트레이드오프Sonnet(36초 타임아웃) → Haiku(3~8초)로 전환해 서비스 가능 수준 달성
🔁
배포 → 실사용 → 버그 발견 → 수정의 반복로컬 테스트로는 발견 못한 iOS 카메라 에러, 타임아웃 등을 실사용에서 발견
